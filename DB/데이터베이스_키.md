# 데이터베이스 키(Key)

# 키 종류표
| 키 종류 | 특징 | 설명 |
|-|-|-|
| 기본키 (Primary Key) | 후보키 중 선택, Not Null | 후보키 중에서 선택된 테이블 대표 키 |
| 대체키 (Alternate Key) | 후보키 중 나머지 | 기본키로 선택되지 않은 후보키 |
| 후보키 (Candidate Key) | 유일성 O, 최소성 O | 튜플을 유일하게 식별할 수 있는 최소한의 속성 집합 |
| 슈퍼키 (Super Key) | 유일성 O, 최소성 X | 튜플을 유일하게 식별할 수 있는 속성 또는 속성들의 집합 |
| 외래키 (Foreign Key) | 다른 테이블의 기본키 참조 | 테이블 간의 관계를 정의하고 참조 무결성을 보장 |

## 데이터베이스 키
키(Key)는 데이터베이스에서 조건을 만족하는 튜블(Tuple, 행)을 찾거나, 순서대로 정렬할 때 다른 튜플과 구별할 수 있는 유일한 기준이 되는 속성(Attribute) 또는 속성의 집합이다.

키는 데이터의 무결성을 보장하고, 효율적인 데이터 관리를 기능하게 한다.

## 키의 주요 특성
- **유일성(Uniqueness)** : 하나의 키 값으로 하나의 튜플만을 유일하게 식별할 수 있어야 한다.
- **최소성(Minimality)** : 유일성을 만족하는 속성들 중 가장 적은 수의 속성으로 구성되어야 한다.

## 기본키 (Primary Key, PK)
후보키 중에서 선택된 단 하나의 키 (테이블을 대표하는 키)

특성:
- NULL 값을 가질 수 없다.
- 중복된 값을 가질 수 없다.
- 테이블 당 오직 하나의 기본키만 존재한다.

선정 기준:
- 보통 길이가 짧고, 값이 자주 변경되지 않으며, NULL 값을 갖지 않는 후보키를 기본키로 선정한다.

## 대체키 (Alternate Key)
기본키로 선택되지 못한 나머지 후보키

특성:
- 후보키의 성질(유일성, 최소성)을 그대로 가진다.

## 후보키 (Candidate Key)
테이블의 각 행을 유일하게 식별하면서 최소한의 속성으로 구성된 키

특성:
- 유일성과 최소성을 모두 만족해야 한다.

## 슈퍼키 (Super Key)
테이블의 각 행을 유일하게 식별할 수 있는 하나의 속성 또는 속성의 집합

특성:
- 유일성을 만족하지만, 최소성은 만족하지 않는다.

## 외래키 (Foreign Key, FK)
한 테이블의 속성 중, 다른 테이블의 기본키(PK)를 참조하는 속성

목적:
- 테이블 간의 관계(Relationship)를 맺고, 데이터의 참조 무결성을 보장하기 위해 사용

특성:
- 참조하는 테이블의 기본키에 존재하는 값이나 NULL 값만 가질 수 있다.
- 외래키 값은 참조하는 테이블의 기본키와 동일한 도메인을 가져야 한다.
